# 社員セットアップ手順（マルチディーラー方式）

対象: 新しく発注システムを使う社員
方式: **発注サイトは全員で1つを共用**。URLの `?dealer=コード` で各自のスプレッドシートに接続される。
社員側にGitHubアカウントは**不要**。

## 全体像

```
サロン様 → 共用サイト(?dealer=自分のコード) → 自分のGAS → 自分のスプレッドシート
                                                ↓
                                        自分のPCの統合ツール
```

---

## 事前準備（ぶんちゃんが1回だけ）

- [ ] **発注シートのテンプレート**を作る：今の発注シートをコピーして、日付シートを全削除、ClientMasterの中身を空に（ヘッダーは残す）、ItemMasterはそのまま
  ※シートをコピーするとGAS（受注処理＋出荷PDF取込）も丸ごと付いてくる
- [ ] **納品書シートのテンプレート**を作る：00_変換マスタ入り・日付シートなし
- [ ] テンプレート2つを「リンクを知っている全員（閲覧者）」か社員個別に共有

---

## 社員ごとの手順（約30分・遠隔フォロー可）

### ① テンプレートをコピー（社員のGoogleアカウントで）
1. 発注シートのテンプレートを開く →「ファイル」→「**コピーを作成**」
2. 名前を「発注サイト（自分の名前）」に
3. 納品書シートのテンプレートも同様にコピー

### ② GASをウェブアプリとして公開（一番技術的な手順。画面共有推奨）
1. 自分の発注シートを開く →「拡張機能」→「**Apps Script**」
2. 右上「**デプロイ**」→「**新しいデプロイ**」
3. 種類の選択（歯車マーク）→「**ウェブアプリ**」
4. 設定：
   - 次のユーザーとして実行 → **自分**
   - アクセスできるユーザー → **全員**
5. 「デプロイ」→ 承認を求められたら許可（「安全ではないページ」警告が出たら「詳細」→「移動」）
6. 表示された **ウェブアプリのURL**（`https://script.google.com/macros/s/.../exec`）をコピーして**ぶんちゃんに送る**

### ③ ぶんちゃんがサイトに登録（1分）
1. `config.js` の `DEALER_API_URLS` に1行追加：
   ```
   ,'yamada': 'https://script.google.com/macros/s/XXXX/exec'
   ```
2. GitHubへpush → 反映（数分）
3. 社員のサロン案内用URLが確定：
   `https://ai-beauty-dealer.github.io/B2B-Order-System/?dealer=yamada`

### ④ シートをツール用に共有
自分の**発注シート・納品書シートの両方**を「共有」で以下に追加：
```
order-auto@my-project-hanbaikanri-488406.iam.gserviceaccount.com
```
権限は**編集者**。

### ⑤ ClientMasterに自分の顧客を登録
- A列: ログインID　B列: パスワード　C列: 得意先名　D列: 種別（直送なら「直送」）　E列: 発注曜日
- ⚠ **ログインIDは必ず自分のコードで始める**（例: 山田さんなら `y-salon01`）
  → 他の社員とIDがかぶると、URLを間違えたサロン様が他人のシートにログインできてしまう事故を防ぐため

### ⑥ 統合ツールのセットアップ
1. ツールのフォルダをコピーしてもらい `start.bat`（またはexe）を起動
2. 設定タブで、**発注シートID・納品書シートID**を自分のものに変更（URLの `/d/` と `/edit` の間）
3. 「💾 設定を保存」→「**🩺 セットアップ診断**」→ 全部✅になればOK
   （❌が出たら画面の指示どおりに直す。だいたい④の共有漏れ）

### ⑦ 動作確認
1. 自分のURL（`?dealer=自分のコード`）を開いて、テスト用のIDでログイン→テスト発注
2. 自分の発注シートに行が入るのを確認（**ぶんちゃんのシートに入らないことも確認**）
3. ツールで「🔄 取り込み」→ 注文が見えたら完了

---

## 運用ルール

- サロン様への案内は**必ず自分の `?dealer=` 付きURL**で（QRコード化推奨）
- サイトの改良はぶんちゃんのpushで**全員に自動反映**（各自の作業なし）
- 出荷PDF取込は各自のスプレッドシートのメニュー「出荷PDF」から（従来通り）
- 欠品管理ファイルは各自のGASメニュー「欠品管理ファイルを作成・確認」で自動作成される

## 【S】今日中に対処すべきもの（火消し）

- **S-1. 顧客名簿の公開漏洩**：リポジトリがPUBLIC。`history_favorites.json` に実在サロン名50件＋導入商品コードが入っており誰でもDL可能。
- **S-2. 共通PW露出**：`generate_line_messages.gs` のコメントに「共通の "actim" になっている想定」。ログインURL・ID体系・共通PWが揃って公開状態。
- **S-3. API無認可**：ログイン後のAPIは clientName 自己申告のみ。GAS URLは公開なので誰でも任意サロン名で発注・キャンセル・変更・履歴閲覧が可能。管理者専用の全サロン一括同期（`sync_all_history_to_favorites`）も無認証。

### Phase 0（即日・半日）
1. リポジトリをPrivate化（GitHub Pages継続は有料。無料なら Cloudflare Pages / Netlify へ移行）
2. `history_favorites.json` を配信から外し、git履歴からも消す（`git filter-repo`）。データはGAS側Favoritesシートに同期済みなので配信不要にできる
3. PWコメント削除＋全サロンPWを個別ランダム化（`generateLineMessages` で再配布）
4. `app.backup.js` / `style.backup.css` / `.DS_Store` を除去

## 【A】データ事故につながるバグ（Phase 1：1〜2週間）

- **A-1. 編集・削除のサロン名照合バグ**：`code.gs:535` `baseClientName = clientName.split(' ')[0]`。半角スペース入りサロン名で他サロンの注文行を巻き込み削除する時限爆弾。
- **A-2. プレフィックス一致の履歴混線**：`row[4] === clientName || startsWith(clientName + ' ')`。「アヴェ」と「アヴェ 北店」のような名前が並ぶと相互に閲覧・キャンセル可能。
- **A-3. キャンセル＝物理削除**：`clearContents()`→全行書き直し。GASタイムアウトでその日の全注文消失リスク。バックアップなし。
- **A-4. XSS**：商品名・カスタム商品名・カート復元データが未エスケープで innerHTML（app.js:361, 909 ほか約15箇所）。特注商品名→シート→マスター編集時再描画でストアドXSS成立可能。
- **A-5. メンテモード素通り**：ログイン時のみ判定。サーバー側 order 処理はフラグ未チェック。

### Phase 1 設計
1. **トークン認証**：login成功時 `Utilities.getUuid()` → `CacheService.put(token, clientName, 21600)`（6h）。全APIで検証、clientNameはトークンから逆引き。管理者アクションはMASTERトークン必須
2. **照合の完全一致化**：得意先名連結（「サロン名 店販」「サロン名 山田様」）をやめ、Ordersシートに「区分」「スタッフ名」列を追加。E列は純サロン名のみ
3. **注文IDのUUID化**：現在はタイムスタンプepoch。I列に orderId 列追加
4. **キャンセル＝論理削除**：F列を「キャンセル」に更新。clearContents全書き直しを根絶
5. **escapeHtml() ユーティリティ**を innerHTML 注入箇所全部に適用（または textContent 化）
6. メンテフラグを handleOrder / handleMultiOrder 冒頭でもチェック
7. handleLogin をスクリプトロックの外へ（現在は発注処理中にログインも30秒待ち）

## 【B】性能・堅牢性（Phase 2）

| 問題 | 現状 | 設計 |
|---|---|---|
| 商品マスタ毎回フルスキャン | 11,000行 getDataRange() | GAS CacheService にJSON保持（100KB/key制限のため分割）。dataVersion連動破棄 |
| 履歴取得が重い | 31日×2シートのフルスキャン | 月別 OrderIndex シート（orderId→シート名・行範囲）を発注時追記 |
| localStorage 5MB限界 | 11,000件JSONで上限近い。QuotaExceededError未処理 | IndexedDB移行。当面try-catchだけでも |
| お気に入り1セルカンマ連結 | セル5万字上限・指数表記ガード3重 | 1行=1サロン1コード正規化 or 1セルJSON。書式@固定で根本解決 |
| fetchタイムアウトなし | GASコールドスタートで無限「接続中...」 | AbortController 30秒＋リトライ案内 |

## 【C】保守性（Phase 3）

1. **clasp導入**：GASデプロイの手作業コピペを廃止（現にcode.gs未コミット差分245行のデプロイドリフトが発生中）。GitHub Actionsで自動化
2. **app.js分割**：ES Modulesで api.js / state.js / render.js / scanner.js / auth.js。ビルド不要のまま `<script type="module">`
3. **バージョン番号の一元化**：現在3箇所手動更新 → config.js に定数化
4. **死にコード削除**：`personal-purchase-check` / `staff-name-container`（JSが参照するがHTMLに存在しない）

## 良くできてる点（変更不要）

- 二重発注ガード（フロントdisabled＋GAS側10秒シグネチャ照合）
- dataVersionキャッシュ無効化＋1時間スロットリング

## やらなくていいこと

React化・DB移行（Supabase等）・PWA化は現規模（50サロン）では過剰。スプレッドシートが台帳を兼ねる運用価値が大きい。

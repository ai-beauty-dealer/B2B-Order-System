# 発注サイトUX改善案（2026-07-04 リサーチ＋コード解析）

状態: **提案のみ・未実装**。対象は美容師さん（サロン様）側の体験。

## リサーチの要点

- B2B受発注のモバイルUXの定番は「**発注履歴からの再発注**」と「迷わないトップ画面」（業界システムMOS等が売りにしている中核機能）
- **PWA（ホーム画面アプリ化）**の実績: Flipkartでコンバージョン+70%・再訪+40%、Alibabaで+76%・再エンゲージ4倍。「アイコンから戻る客は予約率2倍」（Weekendesk）
- 要するに「**開くまでの摩擦**」と「**繰り返し注文の手数**」を削るのが最も効く

## 「たまに重い」の正体（コード解析）

1. **GASのコールドスタート**: しばらく誰も使ってないと初回応答が1〜2秒（Googleの仕様。消せない）
2. **発注確定が重い**: handleOrderが毎回 ①ItemMaster全読み（9,843行、別注判定のため）②その日のシート全読み（二重発注チェック）③グローバルロック（同時発注は順番待ち）→ **朝のピークに詰まる**
3. 履歴読み込み: 31日分のシート走査（アーカイブ導入で軽くなった。定期退避の継続が大事）

## 改善案

### S級（効果大・最優先）

**S-1. PWA化＋完全自動ログイン「アプリ化」**
- manifest.json＋Service Worker＋アイコン → サロン様は「ホーム画面に追加」して**1タップ起動**
- 静的資産をキャッシュ→起動が体感ゼロ秒。ID/PW記憶は既にあるので自動ログインまで繋げる
- 実装規模: 中（サイトのみ。GAS変更なし）

**S-2. 履歴から「🔁 この内容を再注文」ワンタップ**
- 履歴カードに再注文ボタン→内容がそのままカートへ→数量だけ直して確定
- 毎週同じものを頼むサロン様の発注が数十秒で終わる。B2B受発注の定番機能
- 実装規模: 小〜中（app.jsのみ。カート投入ロジックは既存流用）

**S-3. 納期の見える化**
- トップに「本日分の受付は11:00まで／今の注文は◯月◯日出荷」バナー（締め時間ロジックはGASのgetTargetDateStrをクライアント側に移植するだけ）
- 発注確定画面にも「◯月◯日の出荷予定です」
- 美容師さんの一番の不安「いつ届く？」に先回りで答える＝問い合わせ削減
- 実装規模: 小（サイトのみ）

### A級（差がつく）

**A-1. 検索・絞り込み**（2026-07-04 現状確認）
- メーカー(44社)・カテゴリ(18種)のチップ絞り込みは**既に実装済み・本番稼働**（renderManufacturerChips/renderCategoryChips）。データも充実
- ひらがな/カタカナ/全角半角の吸収も実装済み（normalizeForSearch）
- **【2026-07-04 実装】よく頼む順**: 発注履歴から頻度を算出し、絞り込み後の一覧・検索結果で頻出品を上に＋「🕒よく頼む」バッジ（app.js）
- **【2026-07-04 実装】カテゴリのプルダウン**: 新商品追加時の手打ち表記ゆれ防止。GAS関数 setupCategoryDropdown（ItemMaster/TmpNewItemsのC列に▼）。※既存の「カラー」26件は「カラー関連」の分裂なので統合推奨

**A-2. 欠品情報の共有**
- 欠品リスト（GAS管理）をサイトに反映→履歴・商品に「⚠欠品中・入荷待ち」バッジ
- 今はサロン様に欠品が伝わる仕組みがない。「頼んだのに来ない」の不安を先回りで解消＝信頼
- 実装規模: 中（GASに欠品照会action追加＋表示）

**A-3. 「いつものセット」**
- お気に入りを束ねて名前を付けて（例「毎週の定番」）一括カート投入

### B級（磨き込み）

- B-1. 前回頼んだ数量の表示（「前回: 2個」）＋大きい±ボタン
- B-2. 文字サイズ切替（サロンの照明・老眼対応）
- B-3. お知らせ枠の運用強化（Settingsシート連携は実装済み。新商品・キャンペーン告知をツールから更新できるように）
- B-4. バーコード改良（既存計画: ズーム＋BarcodeDetector＋写真保険）→ `バーコード読み取り改良計画.md`

### P級（速度チューニング。「重い」対策）

- **P-1. 別注専用シート方式**（2026-07-04 ぶんちゃん案で確定）: handleOrder等がItemMaster9,843行を毎回全読みしている別注判定を、**別注品だけを集めた小シート「SpecialItems」**を読む方式に変更。
  - キャッシュ案は却下（注文はランダムに届くので10分キャッシュは切れて読み直すだけ＝効果薄）。専用シートなら常に軽く・常に最新
  - 同期の担保（自己修復型）: ①master_updater.gsのsyncMasterItems末尾でSpecialItemsを再生成 ②handleOrderはSpecialItemsが空/無なら本体から自動再生成してフォールバック
  - 発注確定の体感が一番変わる。実装小〜中
- **P-2. 二重発注チェックの範囲縮小**: シート全行→直近60行だけ（10秒以内の重複を見るだけだから十分）。実装小
- **P-3. 履歴の先読み**: ログイン直後に裏で履歴を取得→タブを開いた瞬間表示。実装小
- P-4. 定期アーカイブの習慣化（実装済みの🗄ボタンを2〜3ヶ月ごとに）
- P-5. コールドスタート自体は消せない→スケルトン表示で体感を整える

## 推奨の進め方

1. **第1弾（速度と定番）**: P-1＋P-2＋S-2（再注文）——リスク小さく体感が変わる
2. **第2弾（アプリ化）**: S-1 PWA＋自動ログイン＋P-3
3. **第3弾（安心感）**: S-3 納期表示＋A-2 欠品バッジ
4. A-1/A-3/B級は反応を見ながら

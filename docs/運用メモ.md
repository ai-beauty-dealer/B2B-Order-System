# B2B Order App 運用メモ

## GASデプロイ手順

1. Apps Script の対象プロジェクトを開く。
2. `code.gs` を開き、既存コードを全選択する。
3. ローカルの `backend/code.gs` を全量コピーして貼り替える。
4. 保存する。
5. デプロイ前にエディタ内検索で下記が存在することを確認する。
   - `function doGet(e)`
   - `function doPost(e)`
   - 追加した関数名（例: `isRecentDuplicateOrder`）
6. 「デプロイを管理」から Web アプリの新バージョンをデプロイする。
7. デプロイ後、下記を確認する。
   - `?action=version` が JSON を返す
   - サイトでログインできる
   - 商品一覧と履歴タブが表示できる
8. 異常時は「デプロイを管理」から直前の正常バージョンへ戻す。

## LINE通知A/B切替

発注通知はLINE公式アカウントAを優先し、Aの月間送信可能残数が少なくなったらLINE公式アカウントBへ切り替える。
Discord通知はバックアップとして常時送信する。

### Script Properties

Apps Script の「プロジェクトの設定」→「スクリプト プロパティ」に下記を設定する。

| キー | 用途 |
|---|---|
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE公式アカウントAのChannel Access Token |
| `GROUP_ID` | LINE公式アカウントAの通知先ID |
| `LINE_CHANNEL_ACCESS_TOKEN_B` | LINE公式アカウントBのChannel Access Token |
| `GROUP_ID_B` | LINE公式アカウントBの通知先ID。個人宛ての場合はBの `userId` を入れる |
| `LINE_FALLBACK_THRESHOLD` | AからBへ切り替える残数。未設定なら `20` |
| `DISCORD_WEBHOOK_URL` | Discord常時通知用Webhook |

### 動作ルール

- Aの残数が `LINE_FALLBACK_THRESHOLD` 以下なら、最初からBで送信する。
- 月が変わってAの残数が回復したら、自動でAから送信する。
- AもBも `LINE_FALLBACK_THRESHOLD` 以下になった場合は、Aへ戻して残り枠を使う。
- Aで送信して `429` や上限系エラーが返ったら、Bで再送する。
- Bで送信に失敗した場合は、Aへ再送を試す。
- A/BどちらのLINE送信に失敗しても、Discord通知は送信する。
- LINEの残数確認はMessaging APIの月間送信上限APIと月間送信数APIを使う。

### B通知テスト

1. Bの公式LINEをぶんちゃん個人LINEで友だち追加する。
2. Apps Scriptに最新の `backend/code.gs` を貼り替えて保存する。
3. 関数一覧から `testSecondaryLineNotification` を選ぶ。
4. 実行して、ぶんちゃん個人LINEにテスト通知が届くことを確認する。
5. 成功したら新バージョンでWebアプリをデプロイする。

### A/B切替テスト

実際にAの残数を20以下まで減らさずに、切替ロジックだけを確認する。

1. Apps Scriptに最新の `backend/code.gs` を貼り替えて保存する。
2. 関数一覧から `testLineFallbackSwitchNotification` を選ぶ。
3. 実行する。
4. Bの個人宛てLINEに「A残数しきい値切替テスト」が届くことを確認する。
5. この関数は `LINE_FALLBACK_THRESHOLD` を一時的にA残数+1へ変更し、実行後に元の値へ戻す。

## 二重発注対策

### フロント側

- 確定ボタン押下後は送信中状態にし、再クリックを防止する。
- 送信中は `注文を確定する` / `変更を保存する` を disabled にする。
- 履歴表示は分単位の日時ではなく `orderId` 単位でグループ化する。
- 同じ分に複数注文がある場合も、履歴カードを別々に表示する。
- 編集時の確認画面は「発注内容の変更確認」「変更を保存する」と表示する。

### GAS側

- 新規発注のみ、直前の重複を拒否する。
- 判定条件は下記の完全一致に限定する。
  - 同じサロン名
  - 同じ商品コードと数量の組み合わせ
  - 10秒以内
- 編集保存は重複拒否の対象外にする。
- 重複と判定した場合はシートに書き込まず、下記エラーを返す。

```txt
同じ内容の発注が直前に送信されています。二重発注防止のため、この送信は処理しませんでした。履歴をご確認ください。
```

# B2B Order App 運用メモ

## GASデプロイ手順

1. Apps Script の対象プロジェクトを開く。
2. `code.gs` を開き、既存コードを全選択する。
3. ローカルの `backend/code.gs` を全量コピーして貼り替える。
4. 保存する。
5. デプロイ前にエディタ内検索で下記が存在することを確認する。
   - `function doGet(e)`
   - `function doPost(e)`
   - 追加した関数名（例: `isRecentDuplicateOrder`）
6. 「デプロイを管理」から Web アプリの新バージョンをデプロイする。
7. デプロイ後、下記を確認する。
   - `?action=version` が JSON を返す
   - サイトでログインできる
   - 商品一覧と履歴タブが表示できる
8. 異常時は「デプロイを管理」から直前の正常バージョンへ戻す。

## 二重発注対策

### フロント側

- 確定ボタン押下後は送信中状態にし、再クリックを防止する。
- 送信中は `注文を確定する` / `変更を保存する` を disabled にする。
- 履歴表示は分単位の日時ではなく `orderId` 単位でグループ化する。
- 同じ分に複数注文がある場合も、履歴カードを別々に表示する。
- 編集時の確認画面は「発注内容の変更確認」「変更を保存する」と表示する。

### GAS側

- 新規発注のみ、直前の重複を拒否する。
- 判定条件は下記の完全一致に限定する。
  - 同じサロン名
  - 同じ商品コードと数量の組み合わせ
  - 10秒以内
- 編集保存は重複拒否の対象外にする。
- 重複と判定した場合はシートに書き込まず、下記エラーを返す。

```txt
同じ内容の発注が直前に送信されています。二重発注防止のため、この送信は処理しませんでした。履歴をご確認ください。
```
